# ComfyUI Workspace Setup (Legacy Documentation)

**Note**: This is legacy documentation. See [README.md](README.md) for current architecture.

This workspace uses a **wrapper** `pyproject.toml` that:
1. Reads metadata and tool config from `ComfyUI/pyproject.toml`
2. Pulls dependencies from `ComfyUI/requirements.txt`
3. Adds custom node dependencies from `wrapper_config.toml`
4. Overrides PyTorch CUDA version from `wrapper_config.toml`

## New Architecture (2026-01)

All management scripts moved to Python in `scripts/`:
- **Cross-platform**: Same code on Linux/Windows/Mac
- **Maintainable**: Single implementation, not bash+batch duplicates
- **Testable**: Python scripts easier to debug
- **Clean**: Shell/batch scripts are 3-line wrappers

See [README.md](README.md) for complete documentation.

## Structure

```
ComfyUI_portable/
├── pyproject.toml          # Wrapper (auto-syncs from ComfyUI/requirements.txt)
├── uv.lock                 # Locked dependencies with cu130 PyTorch
├── .venv/                  # Virtual environment (parent level, not in git)
├── ComfyUI/                # Git repository (clean, gets reset during updates)
│   ├── pyproject.toml      # Upstream (removed dependencies, for reference only)
│   └── requirements.txt    # Upstream dependencies (source of truth)
└── *.sh / *.bat           # Management scripts
```

## Why This Structure?

- **Wrapper pattern**: Parent `pyproject.toml` wraps `ComfyUI/requirements.txt`
- **Auto-sync**: When ComfyUI updates dependencies, run `sync_wrapper.sh` to pull changes
- **Override only**: PyTorch cu130 override via `[[tool.uv.index]]`
- **Git-safe**: `.venv` at parent level, never touched by `git reset`
- **Cross-platform**: Identical `uv.lock` works on Linux/Windows

## Linux Usage

### First Time Setup
Already done! Environment is at: `.venv/`

### Run ComfyUI
```bash
./run_comfy.sh sage          # SageAttention (default)
./run_comfy.sh flash         # FlashAttention
./run_comfy.sh pytorch       # PyTorch attention
```

### Update ComfyUI
```bash
./update_comfy.sh
```
- Auto-backups environment
- Pulls latest ComfyUI code
- **Auto-syncs wrapper** from `ComfyUI/requirements.txt`
- Runs `uv sync` to update environment
- Preserves PyTorch cu130 override

### Backup/Restore
```bash
./backup_env.sh              # Creates timestamped backup
./restore_env.sh             # Restores from uv.lock
./restore_env.sh uv.lock.backup_20260125_163000  # Restore specific backup
```

## Windows Usage

### First Time Setup

1. Install uv:

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

2. Open ComfyUI_portable in Windows:
```cmd
cd D:\Apps\ComfyUI_portable    (adjust path)
sync_windows.bat
```

This syncs the exact same environment from `uv.lock`.

### Run ComfyUI
```cmd
run_comfy.bat sage           REM SageAttention
run_comfy.bat flash          REM FlashAttention
```

### Update
```cmd
update_comfy.bat
```

## Key Files

### wrapper_config.toml
**Edit this file** to customize your environment:
```toml
[pytorch]
index_name = "pytorch-cu130"  # Change to cu121, cu124, etc.
index_url = "https://download.pytorch.org/whl/cu130"

[dependencies]
custom = [
    "comfyui-manager>=4.0.5",
    # Add your custom node dependencies here
]
```

### pyproject.toml (Wrapper)
Auto-generated by `sync_wrapper.sh`. Do not edit manually.

### sync_wrapper.sh
Regenerates parent `pyproject.toml` from upstream files + `wrapper_config.toml`:
- Reads `ComfyUI/pyproject.toml` for metadata + tool config
- Reads `ComfyUI/requirements.txt` for dependencies
- Reads `wrapper_config.toml` for custom dependencies + PyTorch override
- Generates final `pyproject.toml`

### uv.lock
Locked versions of all 197 packages including:
- torch==2.10.0+cu130
- All custom node dependencies
- SAM2 from git

## Advantages Over pip

| Feature | pip + requirements.txt | uv + wrapper + lock |
|---------|----------------------|-----------|
| Exact reproducibility | ❌ (version ranges) | ✅ (locked hashes) |
| Cross-platform sync | ⚠️ (can differ) | ✅ (identical) |
| Custom indexes | ⚠️ (via --index-url) | ✅ (in pyproject.toml) |
| Git dependencies | ⚠️ (can update) | ✅ (locked commit) |
| Update safety | ❌ (can break) | ✅ (controlled via lock) |
| Upstream sync | ❌ (manual merge) | ✅ (auto via sync_wrapper.sh) |
| Speed | Slow | Very fast |

## Updating Dependencies

### When ComfyUI updates requirements.txt
```bash
./sync_wrapper.sh  # Regenerate wrapper from ComfyUI/requirements.txt
uv lock            # Update lock with new dependencies
uv sync            # Install updated environment
```

### Add new custom node dependency
Edit `wrapper_config.toml` to add to the custom dependencies list, then:
```bash
./sync_wrapper.sh  # Regenerate wrapper with new dependency
uv lock
uv sync
```

### Change PyTorch CUDA version
Edit `wrapper_config.toml`:
```toml
[pytorch]
index_name = "pytorch-cu121"  # or cu124, etc.
index_url = "https://download.pytorch.org/whl/cu121"
```
Then run:
```bash
./sync_wrapper.sh
uv lock
uv sync
```

### Update specific package
```bash
uv lock --upgrade-package torch
uv sync
```

### Update all packages
```bash
uv lock --upgrade
uv sync
```

## Troubleshooting

### Environment mismatch
```bash
# Linux
./restore_env.sh

# Windows
restore_env.bat
```

### PyTorch version wrong
Check `uv.lock`:
```bash
grep "name = \"torch\"" uv.lock -A 5
```
Should show `version = "2.10.0+cu130"`

### Git conflicts with pyproject.toml
Won't happen - `pyproject.toml` is at parent level, not in ComfyUI git repo.

## Migration from Old Setup

Old location: `ComfyUI/.venv` and `ComfyUI/pyproject.toml` (inside git repo)
New location: 
- `.venv/` at parent level (git-safe)
- `pyproject.toml` at parent level (wrapper)
- `uv.lock` at parent level (cross-platform)

The old environment was moved to `.venv_backup/`. You can delete it once verified working.

## How It Works

1. **Upstream changes**: ComfyUI updates `requirements.txt`
2. **Update script**: Runs `sync_wrapper.sh` after `git pull`
3. **Wrapper regenerates**: Pulls new requirements, adds custom deps, keeps cu130 override
4. **Lock updates**: `uv sync` updates from regenerated wrapper
5. **Environment stable**: PyTorch stays cu130, custom nodes preserved
